# قائمة نقاط النهاية (Endpoints) للمشروع

## 1. مساحة التخزين (Auth)

### 1.1 تسجيل الدخول (Login)
- **المسار**: `/api/auth/login`
- **الطريقة**: `POST`
- **الوصف**: تسجيل دخول المستخدم إلى النظام وتوليد التوكن
- **المعلمات المطلوبة**:
  - `email` (string): البريد الإلكتروني للمستخدم
  - `password` (string): كلمة المرور للمستخدم
- **الاستجابة الناجحة**:
  ```json
  {
    "user": {
      "id": 1,
      "name": "اسم المستخدم",
      "email": "user@example.com"
    },
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
  ```

### 1.2 إنشاء حساب (Register)
- **المسار**: `/api/auth/register`
- **الطريقة**: `POST`
- **الوصف**: إنشاء حساب جديد للمستخدم
- **المعلمات المطلوبة**:
  - `name` (string): الاسم الكامل للمستخدم
  - `email` (string): البريد الإلكتروني للمستخدم
  - `password` (string): كلمة المرور للمستخدم
  - `password_confirmation` (string): تأكيد كلمة المرور
- **الاستجابة الناجحة**:
  ```json
  {
    "user": {
      "id": 1,
      "name": "اسم المستخدم",
      "email": "user@example.com"
    },
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
  ```

## 2. الحجوزات (Reservations)

### 2.1 إنشاء حجز جديد
- **المسار**: `/api/reservation`
- **الطريقة**: `POST`
- **الوصف**: إنشاء حجز طاولة جديد للمستخدم المسجل
- **المصدر**: Token (Bearer) في الهيدر Authorization
- **المعلمات المطلوبة**:
  - `name` (string): الاسم المستخدم للحجز
  - `phone` (string): رقم الهاتف للمستخدم
  - `tables` (number): عدد الطاولات المطلوبة
  - `people` (number): عدد الأفراد
  - `date` (string): تاريخ الحجز (تنسيق YYYY-MM-DD)
  - `time` (string): время الحجز (تنسيق HH:MM)
- **الاستجابة الناجحة**:
  ```json
  {
    "message": "Reservation created successfully",
    "reservation": {
      "id": 1,
      "name": "اسم المستخدم",
      "phone": "0123456789",
      "tables": 1,
      "people": 2,
      "date": "2024-01-20",
      "time": "18:00"
    }
  }
  ```

## 3. التصنيفات والوجبات (Categories & Meals)

### 3.1 الحصول على جميع التصنيفات
- **المسار**: `/api/categories`
- **الطريقة**: `GET`
- **الوصف**: الحصول على قائمة جميع تصنيفات الوجبات
- **الاستجابة الناجحة**:
  ```json
  {
    "data": [
      {
        "id": 1,
        "name": "حلويات"
      },
      {
        "id": 2,
        "name": "سناكس"
      }
    ]
  }
  ```

### 3.2 الحصول على جميع الوجبات
- **المسار**: `/api/meals`
- **الطريقة**: `GET`
- **الوصف**: الحصول على قائمة جميع الوجبات بدون تصنيف محدد
- **الاستجابة الناجحة**:
  ```json
  {
    "data": [
      {
        "id": 1,
        "name": "وجبة 1",
        "price": 50
      },
      {
        "id": 2,
        "name": "وجبة 2",
        "price": 75
      }
    ]
  }
  ```

### 3.3 الحصول على وجبات تصنيف معين
- **المسار**: `/api/categories/{categoryId}/meals`
- **الطريقة**: `GET`
- **الوصف**: الحصول على قائمة جميع الوجبات الموجودة في تصنيف معين
- **المعلمات المطلوبة**:
  - `categoryId` (number): معرف التصنيف (في المسار)
- **الاستجابة الناجحة**:
  ```json
  {
    "meals": [
      {
        "id": 1,
        "name": "وجبة 1",
        "price": 50
      },
      {
        "id": 2,
        "name": "وجبة 2",
        "price": 75
      }
    ]
  }
  ```

## 4. الاشتراكات (Subscriptions)

### 4.1 اشتراك في النشرة الإخبارية
- **المسار**: `/api/subscribe`
- **الطريقة**: `POST`
- **الوصف**: اشتراك المستخدم في النشرة الإخبارية للمطعم
- **المعلمات المطلوبة**:
  - `email` (string): البريد الإلكتروني للمستخدم
- **الاستجابة الناجحة**:
  ```json
  {
    "message": "تم الاشتراك بنجاح"
  }
  ```

## 5. اتصل بنا (Contact)

### 5.1 إرسال رسالة تواصل
- **المسار**: `/api/contact`
- **الطريقة**: `POST`
- **الوصف**: إرسال رسالة تواصل من المستخدم إلى المطعم
- **المصدر**: Token (Bearer) في الهيدر Authorization
- **المعلمات المطلوبة**:
  - `name` (string): الاسم الكامل للمستخدم
  - `email` (string): البريد الإلكتروني للمستخدم
  - `phone` (string): رقم الهاتف للمستخدم
  - `message` (string): نص الرسالة
- **الاستجابة الناجحة**:
  ```json
  {
    "message": "تم إرسال الرسالة بنجاح"
  }
  ```
